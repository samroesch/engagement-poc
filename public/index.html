Architecture POC

Two versions exploring how to load, stream, and render large engagement datasets efficiently.

V1 — Legacy Reference

SQLite + N+1 vs Flat Query

Demonstrates the N+1 waterfall problem and solves it with a single flat JOIN query, virtual scroll rendering, and WebSocket live updates. Uses SQLite with seeded data.

⚠️ Demo runs locally only — requires SQLite binary. Not deployed to Railway.

V2 — Current

PostgreSQL + Progressive Loading

Builds on V1 with a real PostgreSQL database, cursor-based pagination, server-side filtering, dynamic column selection, and a collapsible tree view. 100,000 rows seeded from your production schema.
